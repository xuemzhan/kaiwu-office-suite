# 工程目录规范

本工程面向 Windows 7 SP1 离线交付，目录结构必须稳定、可审计、可打包。

## 根目录规则

根目录只保留人工入口、项目说明和构建入口：`install.bat`、`check.bat`、`repair.bat`、`uninstall.bat`、`verify_installers.bat`、`release_preflight.bat`、`build_zip.bat`、`README.md`、`BUILD.md`。

禁止在根目录新增运行日志、报告、结果、安装状态、临时脚本或复制安装包。

## 标准目录

- `config/`：组件配置，必须是可解析 JSON。
- `docs/`：安装、维护、审查、验收和目录规范文档。
- `manifest/`：版本、软件锁定清单、SHA256 清单。
- `packages/raw/`：唯一安装制品输入目录；正式构建前必须来源审批和哈希校验。
- `scripts/`：安装、检测、修复、卸载、集成、下载和管理脚本。
- `templates/`：离线模板资源。
- `tests/`：非破坏性回归测试。
- `web-app/`：静态导航页，不承担本地程序启动能力。
- `runtime/`：运行时生成目录，不纳入 Git 或正式 ZIP。

## runtime 分区

- `runtime/logs/`：脚本日志。
- `runtime/reports/`：检测、哈希和审查报告。
- `runtime/results/`：集成脚本输出。
- `runtime/state/`：安装状态等本地状态文件。

旧版 `logs/`、`reports/`、`results/`、`state/` 只作为历史兼容忽略目录，不允许新脚本写入。

## Win7 SP1 兼容约束

- 批处理入口保持 `cmd.exe` 兼容语法，不引入 Bash、Python 或新版 PowerShell 作为必需运行时。
- 哈希校验优先使用 Win7 可用的 `certutil`。
- 集成脚本失败时必须返回明确退出码，未验证能力必须返回 unsupported，而不是生成假成功结果。
- 任何 Electron、WPS 插件、PPT 工作流或兼容层组件，必须先进入清单和审查文档，再进入安装链。

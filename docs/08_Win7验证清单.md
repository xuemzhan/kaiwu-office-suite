# Windows 7 SP1 实机验证清单

当前状态：**待执行**。

## 基础安装验收

1. 干净 Windows 7 SP1 64 位虚拟机完成全新安装。
2. 已安装 WPS 的 Windows 7 办公机完成兼容安装。
3. 逐个确认 14 项清单文件及 KexStepup。
4. 验证静默参数、3010 重启码、重启后状态。
5. 运行 `check.bat`，必需组件不得出现 FAIL。
6. 验证 Everything 搜索、Tesseract 中英文 OCR、Obsidian 笔记、Git 状态。
7. 验证修复和卸载，确认用户 Vault 不丢失。
8. 记录机器编号、日志、截图、测试人和日期。

## WPS/Kaiwu 专项验收

1. 确认目标 WPS 版本满足 Kaiwu 上游要求：WPS Office 12.1.0.26375+。
2. 确认插件目录写入 `%APPDATA%\kingsoft\wps\jsaddons\` 后，WPS Writer 能正常加载“开悟”选项卡。
3. 确认 Windows 7 SP1 下插件页面不出现 WebView/JS 运行时错误。
4. 确认 AI 出站 HTTPS、模型密钥、日志脱敏和数据出境审批均已完成。
5. 未通过以上项目时，`config/wps-addon/wps_addon.json` 必须保持 `enable_ai=false`，不得宣传 WPS AI 已交付。

## PPT workflow 专项验收

1. 当前 `scripts/integration/call_ppt_workflow.bat` 是门禁脚本，预期行为为：缺参数退出 1，有参数退出 2。
2. 若要启用真实 PPT 生成，必须先解决上游 Python `>=3.12` 与 Windows 7 SP1 的兼容冲突。
3. 必须锁定并验证 `python-pptx`、`Pillow`、字体、模板、图片渲染、LibreOffice/Poppler/Playwright 等依赖。
4. 必须验证输出 `.pptx` 能被目标机 WPS/PowerPoint 打开，且模板、字体、图片、备注页正常。
5. 未通过以上项目时，不得把 PPT 自动生成/排版写入演示材料或用户手册的已交付能力。

只有基础安装、WPS/Kaiwu 专项和 PPT workflow 专项均按目标能力通过，才能把候选版改为正式版。若某专项不启用，必须在发布说明中明确标为“未交付/门禁返回 unsupported”。

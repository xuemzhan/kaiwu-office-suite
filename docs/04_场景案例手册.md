## 一、案例一：使用 WPS 插件生成汇报材料

### 场景描述

需要根据以下要点生成一段正式汇报材料：
- 本季度完成项目A
- 项目进度达到80%
- 预计下季度完成
- 存在资源不足风险

### 操作步骤

1. **打开WPS Office**
   - 双击桌面上的WPS图标
   - 新建一个空白文档

2. **启用开悟插件**
   - 在菜单栏找到"开悟插件"选项卡
   - 点击"帮我写"按钮

3. **输入主题**
   - 在弹出的对话框中输入：`请根据以下要点生成一段正式汇报材料：本季度完成项目A，项目进度达到80%，预计下季度完成，存在资源不足风险`

4. **生成内容**
   - AI将根据输入生成汇报材料
   - 等待生成完成

5. **审核和修改**
   - 审核生成的内容
   - 根据需要进行修改
   - 保存文档

### 验证要点

1. ✅ WPS插件可调用
2. ✅ 可生成正文
3. ✅ 可润色
4. ✅ 可扩写
5. ✅ 可缩写
6. ✅ 可生成标题

## 二、案例二：使用本地搜索查找项目资料

### 场景描述

需要查找最近一个月修改过的"智能办公"相关材料。

### 操作步骤

1. **打开Everything**
   - 双击桌面上的Everything图标
   - 或使用快捷键 `Ctrl+Shift+F`

2. **输入搜索条件**
   - 在搜索框输入：`智能办公`
   - 按回车键开始搜索

3. **筛选时间范围**
   - 点击搜索框右侧的筛选按钮
   - 选择"修改时间" → "最近一个月"

4. **查看结果**
   - 浏览搜索结果
   - 双击文件可直接打开
   - 右键可选择更多操作

### 验证要点

1. ✅ Agent调用Everything
2. ✅ 返回文件列表
3. ✅ 可打开文件
4. ✅ 可复制文件路径
5. ✅ 可生成材料清单

## 三、案例三：使用 OCR 识别扫描件并入库

### 场景描述

需要识别一张扫描图片，并保存到个人知识库。

### 操作步骤

1. **准备图片**
   - 确保图片清晰可读
   - 图片方向正确

2. **启动OCR工具**
   - 双击桌面上的"OCR识别"图标

3. **选择图片**
   - 点击"选择图片"按钮
   - 选择要识别的图片

4. **选择语言**
   - 根据图片内容选择识别语言
   - 中文图片选择"中文"
   - 英文图片选择"英文"
   - 中英混合选择"中英混合"

5. **开始识别**
   - 点击"开始识别"按钮
   - 等待识别完成

6. **保存到知识库**
   - 点击"保存到知识库"按钮
   - 选择保存位置（如`03_OCR识别`目录）
   - 输入笔记标题
   - 点击保存

### 验证要点

1. ✅ 调用Tesseract
2. ✅ 输出TXT
3. ✅ 输出Markdown
4. ✅ 写入Obsidian Vault
5. ✅ 可被Everything检索

## 四、案例四：使用 Obsidian 沉淀项目知识

### 场景描述

需要把一份材料整理成一篇项目知识卡片。

### 操作步骤

1. **启动Obsidian**
   - 双击桌面上的Obsidian图标
   - 打开Vault：`KaiwuVault`

2. **选择目录**
   - 根据内容选择保存目录
   - 项目资料选择`01_项目资料`
   - 个人知识选择`04_个人知识`

3. **创建笔记**
   - 点击"新建笔记"按钮
   - 输入笔记标题
   - 选择笔记目录

4. **编写内容**
   - 使用Markdown语法编写内容
   - 添加标题、段落、列表等
   - 添加标签和链接

### 验证要点

1. ✅ 自动生成Markdown
2. ✅ 按模板归档
3. ✅ 写入知识库
4. ✅ 生成标签
5. ✅ 生成关联链接

## 五、案例五：使用 Agent 分析安装日志

### 场景描述

WPS插件没有加载成功，需要分析安装日志找出原因。

### 操作步骤

1. **定位日志文件**
   - 打开日志目录：`logs/`
   - 查找安装日志：`install_*.log`

2. **读取日志文件**
   - 使用文本编辑器打开日志文件
   - 搜索关键词："WPS"、"插件"、"错误"、"失败"

3. **分析错误信息**
   - 查看错误代码
   - 查看错误描述
   - 查看上下文信息

4. **查找解决方案**
   - 根据错误信息查找解决方案
   - 参考常见问题FAQ
   - 联系技术支持

### 验证要点

1. ✅ 读取日志
2. ✅ 分析错误
3. ✅ 给出修复建议
4. ✅ 可调用repair脚本
5. ✅ 生成问题报告

## Kaiwu 审查结论

- `package.json` 显示版本 `0.4.0`，插件类型为 `wps`，依赖 `highlight.js`、`html2canvas`、`marked`、`mermaid`，开发依赖包含 `wpsjs`、`playwright-core`、`eslint`、`prettier`。
- `wpsjs.config.js` 显示插件面向 WPS Writer：`type: 'wps'`、`appType: 'wps'`、`systems: ['wps']`。
- README/INSTALL 标注运行要求：WPS Office Personal/Professional `12.1.0.26375+`，操作系统 Windows 10 / Windows 11，AI 能力需要出站 HTTPS 到模型供应商。
- 安装结构写入 `%APPDATA%\kingsoft\wps\jsaddons\`，发布包包含 `install.bat`、`uninstall.bat`、`verify.bat`、`publish.xml` 和 `kaiwu_<version>` 目录。

### 对本项目影响

本项目目标是 Windows 7 SP1 64 位，且当前 WPS 安装包与 Kaiwu 上游要求的 WPS 版本不完全一致。因此 Kaiwu 只能作为来源参考和移植对象，不能在 V1.4.1 中作为“已验证 WPS AI 插件能力”宣传。当前配置已默认关闭 AI，并要求先完成来源审批、模型出口策略和 Win7 实机验证。

## ppt-workflow 审查结论

- `pyproject.toml` 显示项目名 `ppt-workflow`，版本 `1.0.0`，入口命令为 `ppt-workflow` 和 `ppt-workflow-list`。
- 上游要求 `requires-python = ">=3.12"`。
- 必备依赖包含 `python-pptx>=1.0.0`、`Pillow>=10.0`。
- README 标注可选依赖：`markitdown`、`playwright`、`LibreOffice (soffice)`、`Poppler (pdftoppm)`。
- 上游是独立 Python/CLI PPT 生成流水线，不是 WPS 插件，也不是当前套件已有 BAT 集成能力。

### 对本项目影响

Python 3.12+ 与 Windows 7 SP1 目标存在重大兼容风险；Playwright/现代浏览器、LibreOffice/Poppler 也需要逐项锁版本和实机验证。因此 V1.4.1 只新增 `call_ppt_workflow.bat` 作为能力门禁：缺参数返回 1，有参数返回 2 并说明未支持，防止调用方误认为 PPT 生成能力已可用。

## 后续准入条件

1. 如启用 Kaiwu：确认目标 WPS 版本满足上游要求，完成 Win7 SP1 实机加载测试，明确 AI 模型供应商、密钥存储、日志脱敏和数据出站审批。
2. 如启用 ppt-workflow：选择 Win7 可运行的 Python 版本与依赖版本，或提供隔离的非 Win7 执行节点；完成 PPTX 生成、模板套用、字体、图片渲染和错误码测试。
3. 两个上游均不得直接绕过本项目 SHA256 清单、发布审批和 `check.bat` 验收流程。

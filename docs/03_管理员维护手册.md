## 四、检测脚本说明

### check.bat

**功能：** 检测所有组件状态

**使用方法：**
```cmd
check.bat
```

**检测内容：**
1. .NET Framework 4.8 是否安装
2. VC++ Runtime 是否安装
3. WebView2 Runtime 是否安装
4. Git for Windows 是否安装
5. Everything 服务是否运行
6. Tesseract-OCR 是否安装
7. WPS Office 是否安装
8. 工具注册表是否存在
9. 日志目录是否存在

---

## 五、修复脚本说明

### repair.bat

**功能：** 修复常见问题

**使用方法：**
```cmd
repair.bat
```

**修复选项：**
1. 修复环境变量
2. 重建快捷方式
3. 重建工具注册表
4. 重建 Obsidian Vault 模板
5. 重新注册 WPS 插件
6. 重启 Everything 服务
7. 重写 Agent 配置文件
8. 修复日志目录权限
9. 全部修复

## 六、卸载脚本说明

### uninstall.bat

**功能：** 卸载套件组件

**使用方法：**
```cmd
uninstall.bat
```

**卸载策略：**
1. 不删除用户个人文档
2. 不删除用户知识库，除非用户确认
3. 删除快捷方式
4. 删除工具注册表
5. 删除 Agent 配置
6. 可选卸载第三方软件
7. 生成卸载日志
8. 保留回滚记录

## 七、版本锁定策略

1. **所有软件版本锁定** - 不允许自动升级
2. **WebView2 Runtime固定109系列** - 不使用最新版
3. **Git for Windows固定2.46.2** - 不升级
4. **.NET Framework使用4.8离线安装包** - 不使用Web在线安装包
5. **Electron类应用必须验证Win7兼容性** - 确保兼容性

## 八、回归测试

本套件附带基于 Windows 7 SP1 / PowerShell 5.1 验证的 38 用例回归测试套件，部署/升级前后建议运行一次以确认所有集成脚本行为正常。

```powershell
powershell -ExecutionPolicy Bypass -File tests\run_tests.ps1
```

测试覆盖：

- 8 个集成脚本：`call_everything_search` / `call_git_status` / `call_obsidian_note` / `call_tesseract_ocr` / `call_wps_summary` / `call_xmind_outline` / `collect_context` / `open_project_folder`
- 4 个生命周期脚本：`check.bat` / `check_agent.bat` / `repair_agent.bat` / `install_runtime.bat`
- 14 项根环境检测（test_rootcheck）

测试报告输出到 `tests\TEST_REPORT.md`（Markdown 表格，含通过率与失败原因）。详见 `tests/README.md`。
